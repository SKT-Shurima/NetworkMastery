# 传统网络架构演进：从专线到互联网

## 导言

这一章介绍企业网络是如何发展的，以及为什么需要 SD-WAN。

---

## 第一代：专线时代（1990s-2000s）

### 架构特点

```
总部数据中心（北京）
      ↑
      │ 专线（昂贵的租赁线路）
      │
      ├── 上海分公司
      ├── 深圳分公司
      ├── 广州分公司
      └── 其他分公司
```

优点：
- 可靠（电信运营商保证 SLA）
- 安全（专网，不走互联网）

缺点：
- 昂贵（按带宽阶梯计费）
- 部署慢（需要 2-4 周）
- 不灵活（难以扩展）

---

## 第二代：MPLS VPN 时代（2000s-2010s）

### 架构特点

MPLS VPN 在专线上创建虚拟通道，可以更灵活地连接多个分支。

优点：
- 比专线便宜 20-30%
- 比专线更灵活
- ISP 提供 SLA 保证

缺点：
- 仍然昂贵（相比互联网）
- 仍然不够灵活
- 需要运营商支持

---

## 第三代：混合网络时代（2010s-2020s）

### 架构特点

```
                    互联网
                      │
                      │ DIA (Direct Internet Access)
                      │
         ┌────────────┼────────────┐
         │            │            │
    分支 A        分支 B        分支 C
    
同时保留 MPLS VPN 作为备份：
         ┌────────────┬────────────┐
         │            │            │
      MPLS ---------- MPLS ------ MPLS
```

优点：
- 成本和性能的平衡
- 互联网成为主要通道，MPLS 作为备份

缺点：
- 管理复杂（两套网络）
- 无法充分优化（各自为政）

---

## 第四代：SD-WAN 时代（2020s+）

### 架构特点

```
                SD-WAN Controller（集中管理）
                        │
         ┌──────────────┼──────────────┐
         │              │              │
    分支 A         分支 B         分支 C
    ├─ 宽带 A      ├─ 宽带 B      ├─ 宽带 C
    ├─ 宽带 B      ├─ 4G          ├─ 宽带 D
    ├─ 4G          └─ 备用       └─ 4G
    └─ 备用

所有链路智能聚合、加密、优化
```

优点：
- 成本最低（60-80% 节省）
- 性能最好（智能优化）
- 最灵活（易于扩展）
- 管理最简（集中控制）
