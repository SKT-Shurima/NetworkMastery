# SD-WAN 安全设计：零信任架构

## 导言

SD-WAN 的安全不是事后考虑，而是架构的核心。这一章讲解如何在 SD-WAN 中实现安全。

---

## 零信任（Zero Trust）的概念

传统网络的安全模型："信任内网，怀疑外网"

SD-WAN 的安全模型："不信任任何人和东西"

```
传统：             SD-WAN：
边界安全           全面加密
  │                   │
  ├─ 防火墙          ├─ IPSec 隧道
  ├─ DPI             ├─ TLS/mTLS
  ├─ IDS/IPS         ├─ 身份认证
  └─ VPN             ├─ 应用级控制
                     └─ 行为检测
```

---

## SD-WAN 的安全层次

### 1. 链路加密

所有 CPE 间的流量都通过 IPSec/TLS 加密隧道。

### 2. 身份验证

- 设备证书验证
- 用户身份验证
- OTP（一次性密码）

### 3. 策略执行

- 应用级访问控制
- 基于用户的策略
- 设备合规性检查

### 4. 威胁防护

- DPI（深度包检测）
- IPS（入侵防护）
- 威胁情报集成

---

## 最佳实践

1. 启用全加密（IPSec）
2. 实施强身份验证（MFA）
3. 集成云安全网关（Zscaler 等）
4. 定期审计和更新策略
5. 监控异常行为
