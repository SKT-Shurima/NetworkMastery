# 网络性能优化

让网络跑得更快。

## 性能优化技术

```
1. TCP 优化：
   - 增大窗口大小
   - 启用 SACK（选择性确认）
   - BBR 拥塞控制算法

2. 缓冲优化：
   - 发送端缓冲
   - 接收端缓冲
   - 边缘节点缓存

3. 压缩：
   - 数据压缩（gzip）
   - 丢弃冗余
   - 削减冗余

4. 多路传输：
   - MPTCP（多路 TCP）
   - 多链路聚合
   - 并行传输
```

## 关键指标

```
延迟：应该 < 100ms（交互性应用）
丢包：应该 < 1%（可靠性）
抖动：应该 < 30ms（实时性）
带宽：应该能满足业务需求
```

## 性能基线

```
正常情况下建立基线：
- 延迟基线：50ms
- 丢包基线：0.1%
- 带宽利用：60%

异常告警：
- 延迟 > 100ms
- 丢包 > 1%
- 带宽 > 90%
```

这就是为什么 SD-WAN 的监控能力那么关键。
