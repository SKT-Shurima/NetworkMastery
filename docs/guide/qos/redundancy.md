# 网络冗余与高可用

关键网络必须支持故障转移。

## 冗余设计

```
单链路（不可靠）：
总部 ←→ 分支（故障则中断）

双链路（可靠）：
总部 ←→ 分支（ISP-A）
    ↖︎  ↙︎
     分支（ISP-B，故障转移）

完全冗余（最可靠）：
设备冗余 + 链路冗余 + 路由冗余
```

## 故障转移机制

| 技术 | 说明 |
|-----|------|
| HSRP/VRRP | 虚拟网关故障转移 |
| BGP 多路由 | 路由级别故障转移 |
| BFD | 快速故障检测（毫秒级） |

## 高可用指标

```
可用性 = 正常时间 / 总时间

四个 9：99.99% = 年停机 52 分钟
五个 9：99.999% = 年停机 5 分钟
```
