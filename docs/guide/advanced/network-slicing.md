# 网络切片：5G 时代的灵活网络

## 导言

5G 时代，网络不再是"一刀切"的。

==网络切片==允许在同一个物理网络上创建多个虚拟的逻辑网络，每个网络有不同的性能和安全要求。

这一章讲解网络切片的概念和应用。

---

## 第一部分：什么是网络切片？

### 核心概念

```
物理网络（共享基础设施）
  └─ 逻辑切片 1：增强型移动宽带（eMBB）
     ├─ 目标：高速率（1 Gbps+）
     ├─ 延迟：100 ms 可接受
     └─ 应用：视频流、下载
  
  └─ 逻辑切片 2：超可靠低延迟（URLLC）
     ├─ 目标：极低延迟（1 ms）
     ├─ 可靠性：99.99%+ 
     └─ 应用：自动驾驶、远程手术
  
  └─ 逻辑切片 3：大规模物联网（mMTC）
     ├─ 目标：海量连接（百万设备）
     ├─ 低成本、低能耗
     └─ 应用：传感器、智能手表

类比：
  一家医院共享网络
  ├─ 网络切片 A：普通 WiFi（员工上网）
  ├─ 网络切片 B：医学影像（CT、核磁，需要大带宽）
  └─ 网络切片 C：手术室（超低延迟，用于遥操作机器人）
```

## 第二部分：网络切片的实现

### 技术栈

```
物理层资源
  ├─ 无线频谱（分配不同频率给不同切片）
  ├─ 基站资源（CPU、内存）
  └─ 核心网容量

虚拟化技术
  ├─ NFV（网络功能虚拟化）
  ├─ SDN（软件定义网络）
  └─ MEC（边缘计算）

切片编排
  └─ 动态资源分配
     ├─ 监控每个切片的资源使用
     ├─ 根据需求动态调整
     └─ 确保 SLA（服务级别协议）
```

### 动态调整示例

```
时间段 1（工作时间）：
  ├─ eMBB 切片：60% 资源（员工视频会议）
  ├─ URLLC 切片：30% 资源（自动驾驶）
  └─ mMTC 切片：10% 资源（传感器）

时间段 2（晚上）：
  ├─ eMBB 切片：40% 资源（用户减少）
  ├─ URLLC 切片：10% 资源（自动驾驶减少）
  └─ mMTC 切片：50% 资源（夜间检测，传感器活跃）

系统自动调整，无需人工干预
```

## 第三部分：应用场景

### 场景 1：智慧城市

```
同一个城市 5G 网络中：

切片 1：城市交通
  ├─ 低延迟（10 ms）
  ├─ 高可靠性
  └─ 用于车辆间通信、红绿灯协调

切片 2：视频监控
  ├─ 高带宽（4K 视频）
  ├─ 延迟无所谓（实时性不强）
  └─ 用于公共安全监控

切片 3：公众上网
  ├─ 中等带宽
  ├─ 最宽松的 SLA
  └─ 用于游客、路人上网

结果：
  └─ 一个网络满足不同需求的应用
```

### 场景 2：工业 4.0

```
工厂内 5G 网络：

切片 A：机器人控制
  └─ 极低延迟（5 ms）必须保证
  └─ 即使其他应用饥渴，也要给机器人保留资源

切片 B：监控摄像头
  └─ 高分辨率，可容忍较高延迟

切片 C：员工移动设备
  └─ 普通上网，可靠性要求低

优势：
  ├─ 机器人永不失控
  ├─ 成本比传统工业网络低
  └─ 灵活性高（增加新机器时不需要新线路）
```

## 第四部分：与现有技术的关系

### VLAN vs 网络切片

```
VLAN（第 2 层）：
  └─ 在交换机级别隔离流量
  └─ 需要路由器才能跨 VLAN 通信

网络切片（端到端）：
  └─ 从用户设备到核心网，整个路径隔离
  └─ 可以独立的 SLA、安全、成本

关系：
  └─ VLAN 是网络切片的一部分实现
  └─ 网络切片是 VLAN + 路由 + 资源管理的组合
```

---

## 最佳实践

1. **SLA 定义**
   - 每个切片定义明确的 SLA
   - 监控并报告是否满足

2. **资源隔离**
   - 不能让一个切片独占所有资源
   - 但关键切片需要有最低保证

3. **故障隔离**
   - 一个切片故障，不影响其他切片
   - 可能需要多条物理路径

4. **成本模型**
   - 不同切片不同计费
   - 关键应用付费更高
