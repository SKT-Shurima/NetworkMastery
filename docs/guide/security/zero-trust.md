# 零信任安全模型：不信任任何人和东西

## 导言

传统网络安全假设：企业内网是安全的，互联网是危险的。

但现实已改变：员工在家办公，应用分散在云上，内网和外网的边界模糊了。

==零信任==推翻了这个假设：**默认不信任，显式验证每一步**。

---

## 第一部分：零信任核心原则

### 七大核心原则

```
1. 验证身份
   └─ 每个用户都必须身份认证（MFA）
   └─ 不管是公司网络还是家里

2. 验证设备
   └─ 设备必须符合安全标准
   └─ 防病毒最新、补丁完整
   └─ 否则隔离或拒绝

3. 最小权限
   └─ 只给用户做工作所必需的权限
   └─ 不要给"可能需要"的权限

4. 假设被入侵
   └─ 设计时假设已经有攻击者在网络内部
   └─ 微分段、检测、自动隔离

5. 检测和响应
   └─ 持续监控异常行为
   └─ 快速隔离和处理

6. 加密所有数据
   └─ 传输时：TLS/IPSec
   └─ 存储时：AES-256
   └─ 应用层也要加密

7. 审计一切
   └─ 所有访问都有日志
   └─ 可审计可追溯
```

## 第二部分：零信任架构

### 传统 vs 零信任

```
传统架构：
  
  Internet
    ↓
  [边界防火墙]  ← 信任边界
    ↓
  内网
    ├─ PC
    ├─ Server（完全信任，没有微分段）
    ├─ Database
    └─ 文件服务器

问题：
  ├─ 内网 PC 被恶意软件入侵
  ├─ 恶意软件可以访问所有内网资源
  ├─ 无法检测到（没有内网防火墙）
  └─ 完全信任导致灾难

零信任架构：

  Internet
    ↓
  [身份验证 MFA]
    ↓
  [设备检查]
    ├─ 防病毒最新？
    ├─ 补丁完整？
    └─ 符合安全基线？
    ↓
  [应用网关]  ← 真正的访问点
    ├─ 你要访问什么？
    ├─ 你有权限吗？
    └─ 这是异常行为吗？
    ↓
  [微分段]（隔离的网络）
    ├─ PC 只能访问 File Server
    ├─ Web 应用只能访问 Database
    ├─ 其他 PC 无法访问彼此
    └─ Database 无法访问互联网

结果：
  ├─ 即使某台 PC 被入侵
  ├─ 攻击者也只能访问该 PC 被授权的资源
  ├─ 无法横向移动
  └─ 检测系统能识别异常
```

## 第三部分：实施步骤

### 阶段 1：基础设施（第 1-3 个月）

```
□ 部署身份提供商（Identity Provider）
  └─ Azure AD、Okta、Jump Cloud 等

□ 启用 MFA
  └─ 所有员工必须开启多因素认证

□ 部署 ZTNA（零信任网络访问）网关
  └─ 例：Cloudflare Access、Zscaler
  └─ 所有应用都必须通过网关访问

□ 建立设备清单和监控
  └─ MDM（移动设备管理）
  └─ EDR（终端检测和响应）
```

### 阶段 2：应用微分段（第 3-6 个月）

```
□ 映射应用依赖关系
  └─ 哪些应用需要访问哪些数据库

□ 为每个应用创建网络规则
  └─ PC A 可以访问 File Server，仅此而已
  └─ Web 应用可以访问 Database，不能访问其他应用

□ 实施网络微分段
  └─ 使用防火墙、VLAN 或安全组
  └─ 不同分段间的流量受限

□ 验证正常业务不受影响
  └─ 灰度推出
  └─ 先试点部门
```

### 阶段 3：监测和检测（第 6-12 个月）

```
□ 部署 SIEM（安全信息和事件管理）
  └─ 汇总所有日志
  └─ 检测异常行为

□ 配置行为异常检测
  └─ 用户 A 通常 9-18 点工作
  └─ 凌晨 2 点访问超大文件 → 告警

□ 设置自动响应
  └─ 检测到异常 → 自动隔离设备
  └─ 用户确认 → 恢复访问
```

## 第四部分：常见误区

### 误区 1：零信任 = 完全限制

```
错误理解：
  └─ 对用户来说，访问变得困难
  └─ 需要多次认证，经常弹 MFA
  └─ 员工抱怨

正确理解：
  └─ 不是阻止用户，而是验证后放行
  └─ 第一次需要 MFA，之后在信任期内自动通过
  └─ 用户体验应该好于传统 VPN
```

### 误区 2：零信任需要改写所有应用

```
错误理解：
  └─ 太复杂，成本太高

正确理解：
  ├─ 不需要改应用代码
  ├─ 通过网络层和应用层网关实施
  ├─ 应用无感知

例：
  传统：PC → VPN → App
  零信任：PC → [认证/MFA] → [应用网关] → App
         应用本身不变
```

## 第五部分：零信任成熟度模型

```
等级 1：初级
  ├─ 启用了 MFA
  ├─ 有些应用需要身份认证
  └─ 但大多数内网资源仍然可以自由访问

等级 2：中级
  ├─ 所有应用都需要身份认证
  ├─ 启用了设备检查
  ├─ 有部分微分段
  └─ 但某些遗留系统仍然"信任"

等级 3：高级
  ├─ 完整的微分段
  ├─ 所有访问都被记录和监控
  ├─ 异常行为自动隔离
  └─ 定期渗透测试和红队演习

等级 4：成熟
  ├─ 零信任完全融入文化
  ├─ 自动化的安全响应
  ├─ 持续改进
  └─ 业务和安全完全平衡
```

---

## 最佳实践

1. **分步实施**
   - 不要一蹴而就
   - 逐个部门/应用迁移

2. **选择合适的工具**
   - 不一定要最昂贵的
   - 重要是整合和自动化

3. **员工教育**
   - 解释为什么需要零信任
   - 提供支持，避免怨言

4. **持续监控**
   - 零信任是持续的，不是一次性的
   - 定期审视和改进

5. **与业务协调**
   - 安全性和可用性找平衡
   - 不要为了安全而破坏业务
