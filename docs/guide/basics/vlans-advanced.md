# VLAN 进阶：复杂网络的分段隔离

## 导言

前面我们学过基础的 VLAN——按部门或功能把员工分组。但在复杂的企业环络中，VLAN 设计远不止这么简单。

这一章深入讲解 VLAN 的==高级特性==和==设计最佳实践==。

---

## 第一部分：VLAN 的分类

### 按用途分类

```
1. 数据 VLAN（Data VLAN）
   └─ 承载用户的日常业务流量
   
2. 语音 VLAN（Voice VLAN）
   └─ 承载 VoIP 电话的流量（需要特殊优先级处理）
   
3. 管理 VLAN（Management VLAN）
   └─ 只有 IT 人员可以访问，用于管理网络设备
   └─ 与数据网络隔离，防止恶意访问
   
4. 原生 VLAN（Native VLAN）
   └─ Trunk 端口上不带标签的帧属于这个 VLAN
   └─ 通常是 VLAN 1（但要改成别的，不要用默认）
   
5. 隔离 VLAN（Isolated VLAN）
   └─ 用于隔离不信任的设备（如访客 WiFi）
   └─ 设备间不能相互通信
```

### 交换机端口的两种模式

```
模式 1：Access 端口（接入端口）
  └─ 连接终端设备（PC、打印机、电话）
  └─ 只属于一个 VLAN
  └─ 帧不带 VLAN 标签
  
  配置：
    switchport mode access
    switchport access vlan 10

模式 2：Trunk 端口（干道端口）
  └─ 连接到其他交换机或路由器
  └─ 可以承载多个 VLAN
  └─ 帧带有 VLAN 标签（标识属于哪个 VLAN）
  
  配置：
    switchport mode trunk
    switchport trunk allowed vlan 10,20,30,100
```

## 第二部分：VLAN 间路由

### 问题：VLAN 是否能互相通信？

```
答案：不能，除非通过路由器

理由：
  ├─ VLAN 就是把交换机分成多个独立的"小网络"
  ├─ 数据链路层（L2）的通信受限于 VLAN
  ├─ 要跨 VLAN 通信，必须上升到第 3 层（IP 路由）
  
场景：
  VLAN 10（市场部）：10.0.10.0/24
  VLAN 20（技术部）：10.0.20.0/24
  
  市场部的 PC（10.0.10.100）要给技术部的 PC（10.0.20.100）发邮件
  
过程：
  1. 市场部 PC 看到目标 IP（10.0.20.100）在不同网段
  2. 发送给网关（10.0.10.1）
  3. 网关（路由器）接收数据
  4. 路由器在 VLAN 20 接口重新发送
  5. 技术部 PC 接收
```

### 解决方案：使用路由器的子接口

```
传统方法：为每个 VLAN 配置一个物理接口
  ├─ 需要 10 个 VLAN，就需要 10 个物理端口
  └─ 浪费设备资源

现代方法：子接口（Subinterface）
  └─ 一条物理链路，虚拟出多个逻辑接口，每个对应一个 VLAN

配置：

Trunk 端口连接到路由器 Gi0/0
  
  # VLAN 10 的网关（虚拟接口）
  interface Gi0/0.10
  encapsulation dot1Q 10
  ip address 10.0.10.1 255.255.255.0
  
  # VLAN 20 的网关
  interface Gi0/0.20
  encapsulation dot1Q 20
  ip address 10.0.20.1 255.255.255.0
  
  # VLAN 30 的网关
  interface Gi0/0.30
  encapsulation dot1Q 30
  ip address 10.0.30.1 255.255.255.0

结果：
  └─ 只用一条链路，就实现了 3 个 VLAN 的网关功能
```

## 第三部分：VLAN 分配策略

### 策略 1：按部门划分

```
最简单的方法：

VLAN 10：市场部（10.0.10.0/24）
VLAN 20：技术部（10.0.20.0/24）
VLAN 30：财务部（10.0.30.0/24）
VLAN 40：运营部（10.0.40.0/24）
VLAN 100：管理网络（仅 IT）

问题：
  ├─ 如果某个部门搬到另一个楼，需要改交换机端口配置
  ├─ 如果某个部门的人来另一个楼出差，需要改配置
  └─ 管理麻烦
```

### 策略 2：按位置划分

```
VLAN 10：1 楼（10.0.10.0/24）
VLAN 20：2 楼（10.0.20.0/24）
VLAN 30：3 楼（10.0.30.0/24）
VLAN 100：管理网络

优点：
  ├─ 同一楼的设备都在一个 VLAN
  ├─ 即使部门重新调整，VLAN 不需要改
  └─ 交换机配置稳定

缺点：
  ├─ 无法按安全性分组
  ├─ 如果 1 楼有恶意员工，可以嗅探 1 楼的所有流量
  └─ 无法做访问控制（同一 VLAN 内的设备看得到彼此）
```

### 策略 3：按安全级别划分（推荐）

```
VLAN 10：普通用户（员工）
VLAN 20：服务器
VLAN 30：访客/临时用户
VLAN 40：IoT 设备（打印机、摄像头等）
VLAN 100：管理网络（仅 IT）
VLAN 200：隔离 VLAN（被感染的设备可以隔离到这里）

优点：
  ├─ 清晰的安全边界
  ├─ 可以对 VLAN 间的流量设置防火墙规则
  ├─ 即使一个 VLAN 被入侵，也不影响其他 VLAN
  └─ 支持零信任网络模型

缺点：
  └─ 初期设计和规划复杂
```

## 第四部分：VLAN 的常见问题

### 问题 1：错误的原生 VLAN 配置

```
症状：Trunk 端口上有大量不属于任何 VLAN 的流量

原因：
  交换机 1 的 Trunk 端口：Native VLAN = 1
  交换机 2 的 Trunk 端口：Native VLAN = 100
  
  来自交换机 1 的不带标签帧
    → 交换机 2 认为它属于 VLAN 100
    → 被发到 VLAN 100 的设备
    → 问题！

防护：
  ├─ 两个交换机的 Trunk 端口，Native VLAN 必须相同
  ├─ 更好的做法：不使用 VLAN 1 作为 Native VLAN
  └─ 配置：switchport trunk native vlan 999（一个不使用的 VLAN）
```

### 问题 2：VLAN 间通信全部被阻止

```
症状：即使配置了路由，不同 VLAN 间也无法通信

原因：
  ├─ 路由器和交换机之间的 Trunk 端口配置错误
  ├─ 路由器不知道某些 VLAN
  ├─ 防火墙规则默认拒绝 VLAN 间通信
  └─ Native VLAN 设置不一致

诊断：
  1. 确认 Trunk 端口允许相关 VLAN
     show interfaces trunk
     
  2. 确认路由器有所有 VLAN 的子接口
     show ip interface brief | include 0.
     
  3. 测试路由
     ping 10.0.20.1  (从 VLAN 10 的设备)
     
  4. 检查防火墙规则
     show ip access-lists
```

## 第五部分：VLAN 设计最佳实践

### 设计原则

```
1. 分层设计
   └─ 核心交换机（Tier 1）
      ├─ 分布交换机（Tier 2）
      │  ├─ 接入交换机（Tier 3）
      │  │  └─ 终端设备
      └─ 确保高可用性和可扩展性

2. VLAN 数量合理
   └─ 太少（< 5）：安全性差
   └─ 太多（> 50）：管理复杂
   └─ 建议：10-20 个

3. IP 地址规划
   └─ 每个 VLAN 一个子网
   └─ 预留一些 IP 给未来扩展
   └─ 网络号容易识别（如 10.0.10.0/24 表示 VLAN 10）

4. 命名规范
   └─ VLAN 名称反映其用途
   └─ 例：VLAN_OFFICE_TIER1、VLAN_SERVERS、VLAN_GUESTS

5. 文档完整
   └─ 记录每个 VLAN 的用途、IP 段、设备
   └─ 便于故障排查和未来维护
```

## 实战案例：大企业 VLAN 设计

```
场景：1000 名员工的企业，分布在 5 层楼

设计：

VLAN 层级：

  VLAN 10：普通员工（10.0.10.0/24，最多 254 人）
  VLAN 11：普通员工补充（10.0.11.0/24）
  VLAN 12：普通员工补充（10.0.12.0/24）
  VLAN 13：普通员工补充（10.0.13.0/24）
  └─ 共 4 个 VLAN，容纳 1000 人
  
  VLAN 20：服务器（10.0.20.0/24）
    ├─ Web 服务器（10.0.20.10-10.0.20.30）
    ├─ 数据库（10.0.20.50-10.0.20.60）
    └─ 文件服务器（10.0.20.70-10.0.20.80）
  
  VLAN 30：访客（10.0.30.0/24）
    └─ 临时接待，限制网络访问
  
  VLAN 40：设备（10.0.40.0/24）
    ├─ 打印机（10.0.40.10-10.0.40.20）
    ├─ 监控摄像头（10.0.40.50-10.0.40.80）
    └─ IoT（10.0.40.100-10.0.40.150）
  
  VLAN 100：管理（10.0.100.0/24）
    └─ 仅 IT 人员可访问
  
  VLAN 200：隔离（10.0.200.0/24）
    └─ 用于隔离问题设备

防火墙规则示意：

  允许：普通员工 → 服务器
  允许：普通员工 → Internet
  拒绝：普通员工 → 访客 VLAN
  拒绝：普通员工 → 管理 VLAN
  允许：服务器 → Internet（备份和更新）
  拒绝：设备 → 普通员工
  允许：设备 → Internet（固件更新）
  ...（更多规则）

配置复杂度：
  └─ 高（初期费时），但一旦完成，日常维护简单
```

## 最佳实践清单

- [ ] VLAN 数量在 10-20 个之间
- [ ] 每个 VLAN 都有明确的用途
- [ ] IP 地址规划清晰且可扩展
- [ ] Trunk 端口的 Native VLAN 配置正确且一致
- [ ] 路由器配置了所有 VLAN 的子接口
- [ ] VLAN 间的防火墙规则定义明确
- [ ] 文档完整（VLAN 列表、拓扑图、IP 分配表）
- [ ] 定期审计（检查是否有僵尸 VLAN）
- [ ] 隔离 VLAN 已配置（应急用）
- [ ] 管理 VLAN 与数据网络完全分离
